# План лицензионной чистоты для PDF diff highlighter

> **Цель:** привести проект к набору зависимостей и условий распространения, безопасных для коммерческого использования.

**Архитектура:**
- Оставить приложение локальным, без внешней передачи PDF и без облачных сервисов.
- Сначала зафиксировать все прямые и транзитивные зависимости, затем заменить рискованные компоненты на permissive-лицензии.
- Отдельно проверить лицензию кода репозитория, Python-пакетов, NuGet-пакетов и native-артефактов, если они поставляются вместе с продуктом.

**Техстек:**
- Python/Flask-версия в корне репозитория.
- C#-версия в `csharp/`.
- Проверка лицензий через PyPI/NuGet metadata и содержимое пакетов.

---

## Now / Next / Later

- **Now:** проверить и классифицировать все зависимости по риску лицензирования.
- **Next:** заменить проблемные зависимости на permissive-аналоги или убрать их.
- **Later:** зафиксировать итоговую лицензионную политику в `README` и/или отдельном `NOTICE`.

## P0 — Инвентаризация лицензий

### Task 1: Собрать список прямых зависимостей

**Objective:** зафиксировать все прямые зависимости Python и C# версий.

**Files:**
- `requirements.txt`
- `csharp/PdfDiff.csproj`

**Verification:**
- список зависимостей совпадает с тем, что реально используется в коде.

**Acceptance:**
- есть короткая таблица/список с прямыми пакетами и назначением каждого.

### Task 2: Проверить лицензии каждого пакета

**Objective:** для каждого пакета определить тип лицензии и риск для коммерческого распространения.

**Files:**
- внешний источник метаданных PyPI/NuGet
- `requirements.txt`
- `csharp/PdfDiff.csproj`

**Verification:**
- у каждого пакета есть пометка: `safe`, `needs review`, `replace`.

**Acceptance:**
- явно отмечены проблемные пакеты:
  - `PyMuPDF` — AGPL / commercial dual license;
  - `SixLabors.ImageSharp` — split license;
  - `PdfiumViewer` и native PDFium — отдельная проверка на распространение бинарей.

### Task 3: Проверить транзитивные зависимости

**Objective:** убедиться, что транзитивные пакеты не добавляют скрытых ограничений.

**Files:**
- lockfiles, если появятся
- package metadata

**Verification:**
- список транзитивных зависимостей собран и просмотрен.

**Acceptance:**
- отсутствуют неожиданные copyleft-зависимости в цепочке.
